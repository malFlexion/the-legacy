# Final Project Rubric (100 Points)

## Grading Breakdown

### I. Model & Inference (40 Points)

#### Model Functionality (20 pts)
- **Excellent (16-20)**: Model trained, running, producing meaningful outputs. Includes LoRA if trained. Thoughtful evaluation with custom eval dataset.
- **Good (12-16)**: Model runs with limitations. LoRA attempted. Some evaluation.
- **Fair (8-12)**: Significant issues. LoRA unsuccessful. Minimal evaluation.
- **Poor (0-8)**: No functional model.

#### Innovation & Creativity (20 pts)
- **Excellent (16-20)**: Clear stretch of skills. Understands current state of the field and pushes boundaries.
- **Good (12-16)**: Basic innovation, mostly within comfort zone.
- **Fair (8-12)**: Limited creativity, stayed in comfort zone.
- **Poor (0-8)**: No innovation shown.

### II. Production Environment & Programmability (30 Points)

#### Environment Setup (15 pts)
- **Excellent (12-15)**: Model deployed in production environment (local, AWS, etc.). Correctly configured, accessible via API endpoint.
- **Good (8-12)**: Partially successful, needs manual intervention.
- **Fair (4-8)**: Significant difficulties.
- **Poor (0-4)**: No functional environment.

#### Inference Pipeline (15 pts)
- **Excellent (12-15)**: Robust, efficient pipeline. Well-documented, optimized. Includes appropriate sampling method.
- **Good (8-12)**: Basic pipeline with limitations. Basic sampling.
- **Fair (4-8)**: Rudimentary, limited functionality.
- **Poor (0-4)**: No functional pipeline.

### III. Documentation & Presentation (30 Points)

#### Technical Documentation (15 pts)
- **Excellent (12-15)**: Comprehensive docs covering model, pipeline, API, deployment, sampling method.
- **Good (8-12)**: Good docs with some gaps.
- **Fair (4-8)**: Rudimentary.
- **Poor (0-4)**: Little to none.

#### Demo & Presentation (15 pts)
- **Excellent (12-15)**: Clear, concise, engaging demo.
- **Good (8-12)**: Adequate with some explanation.
- **Fair (4-8)**: Basic with limited explanation.
- **Poor (0-4)**: Unclear or incomplete.

## What You Need to Deliver
1. A trained/finetuned model (with LoRA) and evaluation results
2. A deployed, accessible API endpoint
3. A robust inference pipeline with proper sampling
4. Comprehensive technical documentation
5. A clear demo/presentation

## Key Success Factors
- Push beyond your comfort zone (innovation points are 20% of the grade)
- Evaluation is critical - create and use a proper eval dataset
- Documentation quality matters as much as code (30% of grade)
- End-to-end functionality: training -> deployment -> serving -> demo
